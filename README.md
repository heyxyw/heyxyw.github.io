### 某省教育厅人才培养平台-就业信息子系统（2016/01-2016/11）

#### 主要职责：
    1.协助项目经理到客户现场了解客户需求并进行需求分析，制定数据库模型和开发方案，并负责整个系统的开发任务。
    
    2.与客户的第三方厂商对接VPN平台应用。
    
    3.项目新成员的培训。
    
    4.配合项目经理提供验收相关技术支持。
    
    5.编写用户手册，进行用户培训，并持续维护。
#### 开发环境:
        开发工具使用Eclipse, 服务端采用Spring3+SpringMvc3+Hibernate3 。数据库采用Oracle 11g , 服务器使用Tomcat 7.x ，使用Redis作为应用缓存。前端页面使用FreeMarker 、AJAX、JQuery等技术 。

#### 详细内容：
    1. 对基础框架进行账户结构重构以适应本项目需求。
    
    2. 引入Activiti实现流程的审批，把业务数据跟审核流程进行分离。尽可能的减少审核业务数据与流程的耦合，减少了由于流程变化带来的巨大工作量。开发效率大为提升。
    
    3. 针对项目中需要进行大量基础数据字典的读取，我们针对这部分改动量极小的数据做了一层缓存。最开始使用系统缓存，由于数据会一直存在应用中，会占用不必要的内存。然后引入了Redis来作为缓存。在解放了系统压力的同时也提升了系统性能。
    
    4. 针对系统中常用的生源批量导入功能。在原始方案中使用固定模板的方式导入，并且后台以拼装sql 的方式进行数据插入。在此过程中，对数据的校验存在困难，数据导入的效率较低，错误提示不精确，用户体验极差。针对以上问题，我参考PL/SQL 工具在导入数据库时的方式。前台页面让用户对其导入的数据列与系统字段进行匹配。如果列明与系统字段名一致，系统将自动匹配。导入后，后台通过字段与列的对应关系，通过反射的方式对数据进行封装，并通过校验算法对数据进行校验。完成后的正确数据同意批量差异更新，错误数据进行相关位置提示，并下载错误数据文件。用户针对错误数据进行修改后，即可再次进行导入。极大的改善了用户体验并提高了操作效率。
    
    5. 由于生源数据会存在多次的导入，需要我们对数据进行差异更新。如果每次导入都直接去查询哪些数据需要更新哪些数据需要新增，此方式效率极为低下。目前的方案是通过构建一张与主表同结构的临时表，导入数据首先进入临时表中，然后通过数据库层面上去做差异更新的方式来解决。
    
    6. 针对不同的就业率统计类型实现不同的统计，制作相关统计下载模板并提供统计报表的下载！下载文件可直接进行打印装订成册，无需再手动调整格式。
  
    7. 系统提供各学历层次的报到证与派遣证的单个与批量打印。
